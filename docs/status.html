Componentes do sistema

Componente Tipo Descrição Endpoints / API Status
KCP SDK Library Biblioteca cliente. Publica, busca, verifica, gerencia lineage. Funciona in-process sem servidor. Python, TypeScript, Go. node.publish() node.search() node.lineage() ✅ Live
KCP Node (HTTP) Server API REST que expõe o SDK via HTTP. Permite acesso por outros processos, browsers e peers remotos. GET /kcp/v1/artifacts POST /kcp/v1/artifacts GET /kcp/v1/health ✅ Live
MCP Bridge Bridge Expõe o KCP como servidor MCP (Model Context Protocol). Permite que Claude, Cursor e Windsurf publiquem e recuperem artefatos diretamente via tools. MCP tools: kcp_publish kcp_search kcp_get kcp_lineage kcp_list kcp_stats ✅ Live
Sync Engine Background Worker que entrega artefatos para peers registrados. Fila SQLite-backed, retry com backoff exponencial, circuit breaker por peer, confirmação por ACK. POST /kcp/v1/sync/push GET /kcp/v1/sync/status GET /kcp/v1/sync/list ✅ Live
Peer Discovery Protocol Dois mecanismos: bootstrap via peers.json oficial + gossip consultando /kcp/v1/peers de cada peer conhecido (1 hop). Todos os peers descobertos são upsertados localmente. GET /kcp/v1/peers POST /kcp/v1/peers GET /kcp/v1/peers/discover ✅ Live
Replication Tracker Protocol Rastreia quais peers confirmaram (ACK) recepção de cada artefato. Replication factor configurável. Flag complete quando count ≥ factor. GET /kcp/v1/artifacts/{id}/replication ✅ Live
Web Dashboard UI SPA vanilla JS servida pelo próprio nó em /ui. Abas: Artifacts, Search, Publish, Peers, Sync, Replication. Sem dependências externas. GET /ui ✅ Live
ACL Engine Security Controle de acesso por artefato. Visibilidades: public (todos), org (mesmo tenant), team (lista explícita), private (owner only). Verificado em todos os endpoints. Via headers X-KCP-User-ID X-KCP-Tenant ✅ Live
Peer Network (peer04/05/07) Infrastructure 3 nós públicos live, cada um com DB e filesystem independentes. TLS via Let's Encrypt. nginx com rate limiting por zona (leitura / sync / escrita). https://peer04.kcp-protocol.org peer05 peer07 ✅ Live
Multi-region Peers (peer01/02/03/06) Planned Nós em Europa, Américas, Ásia-Pacífico e África. Cada um em host independente com Ansible automatizado. – ⏳ Fase 3
IPFS Backend Planned Armazenamento de conteúdo via IPFS. CID-addressed. Artefatos públicos permanentes, resistentes à censura. – ⏳ Fase 4
